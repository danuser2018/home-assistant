# ADR-005: Distribución mediante Imágenes Precompiladas en DockerHub

## Fecha
28-06-2026

## Estado
Aceptado

## Contexto
El ecosistema de NOVA-2 consta de múltiples microservicios con dependencias pesadas e inferencia de modelos neuronales (STT, TTS). Si obligamos a los usuarios y colaboradores a clonar individualmente cada repositorio de código fuente y a compilar las imágenes locales de Docker en su propia máquina, el tiempo de instalación inicial se dispararía a más de una hora, requiriendo un alto consumo de recursos de CPU/RAM y aumentando la tasa de errores de compilación por inconsistencias del compilador o diferencias de hardware.

## Decisión
Empaquetaremos y distribuiremos todos los microservicios basados en contenedores de NOVA-2 a través de imágenes Docker precompiladas en un registro centralizado público (DockerHub) bajo la organización `danuser2018/`. 

El repositorio principal `home-assistant` contendrá el archivo `docker-compose.yml` que orquesta todo el sistema referenciando directamente a estas imágenes precompiladas (`danuser2018/interaction-manager:latest`, `danuser2018/stt-capability:latest`, etc.). Durante el despliegue local, Docker Compose descargará automáticamente las capas listas para ejecutar.

## Alternativas consideradas
- **Compilación Local de Docker (Docker Build en Host):** Configurar `docker-compose.yml` para compilar localmente apuntando a los repositorios de código clonados. Descartado por el tiempo excesivo de descarga de dependencias pesadas (como paquetes de PyTorch, ONNX y librerías C++) y la ineficiencia de compilar lo mismo en cada instalación.
- **Instalación Nativa vía Script Local (Sin Docker):** Usar scripts bash para instalar Python, entornos virtuales y dependencias en el host para cada servicio de procesamiento. Descartado porque rompe el principio de aislamiento, ensucia el sistema del usuario y genera colisiones entre librerías Python y compiladores del sistema.

## Consecuencias
+ **Instalación ágil e inmediata:** Permite levantar el ecosistema completo en menos de 10 minutos mediante `docker compose up -d`.
+ **Homogeneidad garantizada:** Todos los usuarios ejecutan exactamente el mismo binario de contenedor que ha sido validado previamente, eliminando los fallos del tipo "funciona en mi máquina".
+ **Cero recursos de compilación:** Ahorra CPU, memoria y almacenamiento en el host del usuario durante el aprovisionamiento.
- **Dependencia de Red y Terceros:** Requiere conexión a internet para descargar las imágenes y depende de la disponibilidad del registro DockerHub.
- **Necesidad de Pipeline de Publicación:** Los desarrolladores deben ejecutar un paso de compilación y subida (`docker push`) de imágenes a DockerHub tras realizar cambios de código para que estos se distribuyan al resto de usuarios.
