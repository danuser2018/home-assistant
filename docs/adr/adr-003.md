# ADR-003: Scoring Determinista de Plugins de Intenciones

## Fecha
28-06-2026

## Estado
Aceptado

## Contexto
El servicio `orchestrator` es responsable de recibir el texto transcrito de la voz del usuario y determinar qué acción o plugin específico debe ejecutarse en respuesta (por ejemplo, consultar el tiempo, reportar capacidades, saludar, etc.). Esta resolución de intención debe ejecutarse de forma ágil, con un consumo mínimo de recursos y garantizando respuestas predecibles y repetibles en hardware local modesto (como microcomputadores o PCs de baja potencia).

## Decisión
Utilizaremos un modelo de enrutamiento basado en **puntuación matemática determinista** (*scoring*). 

Cada plugin disponible en el sistema definirá estáticamente sus propias reglas de emparejamiento usando:
- **Palabras clave (Keywords):** Palabras con un peso asociado (por ejemplo, `"tiempo": 5`, `"clima": 5`).
- **Expresiones regulares (Regex):** Patrones de texto específicos con un peso mayor para capturar intenciones más complejas.

El `orchestrator` evaluará el texto normalizado frente a las reglas de todos los plugins cargados. El plugin con la puntuación acumulada más alta, siempre que supere un umbral de puntuación mínimo configurado, será seleccionado para su ejecución. En caso de empate o si ningún plugin supera el umbral, se delegará la respuesta a un `FallbackPlugin` controlado.

## Alternativas consideradas
- **Enrutamiento basado en Modelos de Lenguaje (LLMs locales):** Utilizar modelos de lenguaje pequeños (como Llama 3 o Phi 3 a través de Ollama) para inferir semánticamente la intención. Descartado por la alta latencia introducida (segundos frente a milisegundos), alto consumo de memoria de GPU/RAM, costes computacionales y falta de determinismo (riesgo de alucinaciones o respuestas inconsistentes ante frases similares).
- **Clasificador NLP Estadístico (Naive Bayes / SVM):** Entrenar un modelo de clasificación local. Descartado por añadir complejidad operativa innecesaria (necesidad de reentrenar el modelo cada vez que se añade un plugin) y no ofrecer un control de prioridad de coincidencia tan granular y directo para el desarrollador como las expresiones regulares.

## Consecuencias
+ **Rendimiento excepcional:** La resolución de intenciones toma menos de 5 milisegundos en CPU.
+ **Determinismo absoluto:** Ante la misma frase del usuario, el sistema siempre responderá con el mismo plugin, facilitando el diseño de pruebas y la estabilidad.
+ **Extensibilidad trivial:** Añadir una nueva funcionalidad es tan simple como crear un plugin y asignarle palabras clave representativas, sin necesidad de entrenamientos ni APIs de inferencia.
+ **Cero coste computacional:** No requiere hardware de GPU para resolver la intención.
- **Rigidez semántica:** No posee comprensión de lenguaje natural avanzada. El usuario debe usar frases que contengan las palabras clave configuradas o sigan los patrones de las expresiones regulares para activar el plugin adecuado.
- **Conflictos de coincidencia:** Si dos plugins definen palabras clave similares, pueden competir y requerir un ajuste fino manual de sus pesos de enrutamiento.
